# AGENTS.md

Voice and prose conventions for AI agents writing in this repo.

Write in a terse engineering voice.

Sound like a practical technical writer with the clarity of Kernighan,
the directness of Rob Pike, and the compression of Derek Sivers.

Use plain words. Use active voice. Prefer commands and concrete examples.
Avoid flourishes, slogans, rhetorical reversals, decorative rhythm, and
LLM-style connective tissue.

Do not use em dashes. Do not write in threes for cadence. Do not say
"it's worth noting", "moreover", "in conclusion", or "let's".

If a sentence can be cut, cut it. If an example can replace an abstraction,
use the example. If the answer is done, stop.
